# Gomoku Client v0.0.1 進度檢查清單

最後更新：2026-02-20
維護原則：每完成一個項目，必須同一個 commit 同步更新此清單（勾選狀態 + commit hash）。

## 使用規則

1. 一個可 review 的功能增量，對應一個 commit。
2. 同一個 commit 內必須同時包含：
   - 功能或修正代碼
   - 本清單狀態更新
3. commit message 使用既有格式：`[Type][Scope] Short description`。
4. 每個已完成項目後方補上 commit hash，例如：`(commit: 6594161)`。
5. 每次推進後至少執行一次 `bun run build`，失敗不得勾選完成。

## 跨 Repo 執行順序（Client / Server）

> 這段是 v0.0.1 的實作節奏 gate，不是建議。

1. 先完成 Client 單機可玩基線（至少達成 `C-BOARD-006`、`C-GAME-004`、`C-UX-001`）。
2. 達成 `C-AI-001` 後，立即啟動 Server 基礎骨架（FastAPI app + health check + WebSocket endpoint 雛形），不得延到 AI 全部完成才開始。
3. 在開始 `C-AI-003`（Minimax 深度 2）之前，Server 必須已可本機啟動並可建立基本 WS 連線（即使房間功能尚未完整）。
4. 進入 `C-NET-001` 後，Client 與 Server 進入並行迭代；任何 WebSocket message schema 變更，兩邊必須在同日同步 commit。
5. 若 Server 進度落後，Client 可先推進 `C-AI-004`~`C-AI-009`，但不得阻塞 `C-NET-001` 之前的介面定義工作。
