# Gomoku Client v0.0.1 進度檢查清單

最後更新：2026-02-20
維護原則：每完成一個項目，必須同一個 commit 同步更新此清單（勾選狀態 + commit hash）。

## 使用規則

1. 一個可 review 的功能增量，對應一個 commit。
2. 同一個 commit 內必須同時包含：
   - 功能或修正代碼
   - 本清單狀態更新
3. commit message 使用既有格式：`[Type][Scope] Short description`。
4. 每個已完成項目後方補上 commit hash，例如：`(commit: 6594161)`。
5. 每次推進後至少執行一次 `bun run build`，失敗不得勾選完成。
